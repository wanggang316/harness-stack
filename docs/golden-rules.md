# Golden Rules

harness-stack 的核心原则。所有 Skills 和 Agents 必须遵守。

## 1. AGENTS.md 是地图，不是手册
- 保持在 100-150 行以内
- 只包含导航信息，不包含详细内容
- 详细内容放在 docs/ 和 skills/ 中

## 2. 仓库知识即真理源
- 所有知识版本化在仓库中
- Agent 无法访问的信息 = 不存在
- 文档与代码共同演进

## 3. 一切都是 Skill 或 Subagent
- 没有外部 CLI 工具
- 初始化、验证、评分都是 Skills
- 专业判断交给 Subagents

## 4. 渐进式披露
- 短入口 → 按需加载深度内容
- 避免上下文污染
- 每个文件只包含必要信息

## 5. 证据优先
- 每个验证步骤需要具体证据
- "我检查了" 不够，需要 "我运行了 X，结果是 Y"
- 退出标准必须可验证

## 6. 反合理化
- 每个 Skill 预先列出 Agent 可能的借口
- 提供反驳和正确做法
- 防止 Agent 偷懒跳步

## 7. 平台无关
- Skills 不硬编码框架命令
- 读取项目配置，不做假设
- 询问用户一次，持久化答案

## 8. 增量交付
- 薄垂直切片优于水平分层
- 每个切片都是可工作的功能
- 小步快跑，频繁验证

## 9. 机械化执行
- 能自动化的不靠文档
- 用 hs-check 验证结构
- 用 hs-score 量化质量

## 10. 自举能力
- harness 可以用自己的 Skills 改进自己
- hs-skill-create 创建新 Skills
- hs-check 验证自身质量

## 11. 修复环境，不修复提示
- 当 Agent 反复犯错，改善环境而非重复提示
- 添加 linter 规则、更新 golden rules
- 让正确的做法成为最容易的做法

## 12. 上下文是稀缺资源
- 每个加载到 Agent 上下文的内容都有成本
- 只加载当前任务需要的信息
- 支持文件按需加载，不预加载
