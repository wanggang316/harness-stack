# Design Docs

Design docs 记录 feature 与系统背后的 **why** 与 **how**。

## When to Write a Design Doc

- 触及多个 domain 的新 feature 或新系统
- 重大架构变更
- 带有长期影响的技术决策
- 存在多种方案、且选择本身重要的任何工作
