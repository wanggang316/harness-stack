Load and execute the hs-design skill for creating design documents.

Read the skill definition at skills/hs-design/SKILL.md and follow its Process section step by step.

Steps:
1. Analyze the problem space — understand constraints, identify candidate approaches, surface assumptions
2. Write a design doc covering: Problem Statement, Goals, Non-Goals, Proposed Solution, Alternatives Considered, Dependencies, Risks
3. Present design doc for human review — do NOT proceed until approved
4. Save design doc to docs/design-docs/
