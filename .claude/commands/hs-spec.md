Load and execute the hs-spec skill for spec-driven development.

Read the skill definition at skills/01-define/spec/SKILL.md and follow its Process section step by step.

Steps:
1. Surface assumptions immediately — list what you're assuming and ask for confirmation
2. Write a spec covering 6 core areas: Objective, Commands, Project Structure, Code Style, Testing Strategy, Boundaries
3. Reframe vague requirements as concrete success criteria
4. Present spec for human review — do NOT proceed until approved
5. Save spec to a file in the repository
6. Keep the spec alive — update when decisions or scope change
